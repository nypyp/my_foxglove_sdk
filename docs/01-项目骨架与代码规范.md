# 第一章：项目骨架与代码规范

## 1.1 CMake 项目结构

### 为什么使用 FetchContent

我们使用 CMake 的 `FetchContent` 模块来管理依赖，而非传统的 `find_package` 或 Git Submodules。这样做的好处：

1. **自包含**: 用户无需手动安装依赖
2. **版本锁定**: 通过 `GIT_TAG` 精确控制依赖版本
3. **构建集成**: 依赖与项目一起构建，确保 ABI 兼容

```cmake
# cmake/dependencies.cmake
include(FetchContent)

FetchContent_Declare(
  tl-expected
  GIT_REPOSITORY https://github.com/TartanLlama/expected.git
  GIT_TAG        v1.1.0
)

FetchContent_Declare(
  Catch2
  GIT_REPOSITORY https://github.com/catchorg/Catch2.git
  GIT_TAG        v3.5.2
)

set(EXPECTED_BUILD_TESTS OFF CACHE BOOL "" FORCE)
FetchContent_MakeAvailable(tl-expected Catch2)
```

### 项目 CMakeLists.txt

顶层的 `CMakeLists.txt` 组织库、测试和示例：


## 1.2 错误处理：为什么使用 tl::expected

### 对比异常 (Exceptions)

| 特性 | 异常 | tl::expected |
|------|------|--------------|
| 错误可见性 | 隐藏，需要文档说明 | 显式，在类型中体现 |
| 性能 | 异常路径有开销 | 零开销抽象 |
| 确定性 | 调用栈展开不确定 | 完全确定 |
| 嵌入式 | 通常禁用异常 | 完全兼容 |

Foxglove SDK 选择 `tl::expected` 是因为：
1. C++17 兼容（std::expected 是 C++23）
2. 显式错误处理，不会遗漏错误检查
3. 性能可预测，适合实时系统

### 官方 SDK 的设计参考

官方 `foxglove-sdk/cpp/foxglove/include/foxglove/error.hpp`：

```cpp
enum class FoxgloveError : uint8_t {
  Ok,
  Unspecified,
  ValueError,
  // ... 更多错误码
};

template<typename T>
using FoxgloveResult = tl::expected<T, FoxgloveError>;
```

我们的简化版本：

```cpp
// include/foxglove/error.hpp
enum class FoxgloveError {
  None = 0,
  InvalidArgument = 1,
  ChannelClosed = 2,
  ServerStartFailed = 3,
  IoError = 4,
  SerializationError = 5,
  ProtocolError = 6,
};

template <typename T>
using FoxgloveResult = tl::expected<T, FoxgloveError>;
```

### 与官方实现的对比

| 方面 | 官方 SDK | 我们的实现 |
|------|----------|------------|
| 错误码数量 | 16+ 个（完整功能） | 7 个（核心必需） |
| 底层类型 | `uint8_t` | 默认 `int` |
| 字符串函数 | `strerror()` 返回 `const char*` | `foxglove_error_string()` 返回 `std::string` |
| 警告系统 | `WarnStream` 类 | 本章暂不实现 |

## 1.3 FOXGLOVE_TRY 宏

手动展开 `tl::expected` 的错误检查很繁琐：

```cpp
// 没有宏时
auto result = some_operation();
if (!result.has_value()) {
  return tl::make_unexpected(result.error());
}
auto value = result.value();
```

`FOXGLOVE_TRY` 宏简化为：

```cpp
FOXGLOVE_TRY(value, some_operation());
// 如果出错，自动 return；否则 value 绑定到结果
```

实现原理：

```cpp
#define FOXGLOVE_TRY(var, expr)                                      \
  auto _foxglove_try_##var = (expr);                                 \
  if (!_foxglove_try_##var.has_value()) {                            \
    return tl::make_unexpected(_foxglove_try_##var.error());         \
  }                                                                  \
  auto var = std::move(_foxglove_try_##var.value())
```

宏使用 `##` 拼接生成唯一变量名，避免命名冲突。

## 1.4 代码规范：.clang-format

我们采用 Google 风格作为基础，配合以下定制：

```yaml
---
Language: Cpp
Standard: Latest
BasedOnStyle: Google

# 缩进
IndentWidth: 2
TabWidth: 2
UseTab: Never
NamespaceIndentation: None

# 行长度
ColumnLimit: 100

# 花括号
BreakBeforeBraces: Attach

# 函数和 Lambda
AllowShortFunctionsOnASingleLine: Empty
AllowShortLambdasOnASingleLine: Empty

# 指针对齐
PointerAlignment: Left

# 包含排序
SortIncludes: true
IncludeCategories:
  - Regex: "^<foxglove"      # 本项目头文件优先
    Priority: 1
  - Regex: "^<.*/"            # 第三方库
    Priority: 2
  - Regex: "^<.*"             # 标准库
    Priority: 3
  - Regex: ".*"               # 其他
    Priority: 4
```

## 1.5 测试框架：Catch2 v3

### 测试结构

```cpp
// tests/test_error.cpp
#include <foxglove/error.hpp>
#include <catch2/catch_test_macros.hpp>

TEST_CASE("FoxgloveError - enum values match spec") {
  SECTION("None is zero") {
    REQUIRE(static_cast<int>(foxglove::FoxgloveError::None) == 0);
  }
  // ...
}
```

### 命名规范

- **测试文件**: `test_<module>.cpp`
- **测试用例**: `TEST_CASE("ClassName - behavior description")`
- **分区**: `SECTION("specific scenario")`

### TDD 工作流

我们严格遵循测试驱动开发：

1. **红**: 编写测试，确认编译/运行失败
2. **绿**: 编写最小实现，使测试通过
3. **重构**: 清理代码，保持测试通过
